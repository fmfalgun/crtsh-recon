> WHAT IS CRT.SH

Certificate Transparency is a public ledger where every TLS certificate must be logged before browsers trust it. Google mandated it in 2018. Every cert issued for *.yoursite.com or any subdomain is recorded — permanently, publicly.

crt.sh is Sectigo's free search interface to these logs, exposing a JSON API: crt.sh/?q=%.domain&output=json

Because cert issuance happens before deployment, CT logs reveal subdomains not yet live, staging environments, and infrastructure shared across unrelated domains.

> WHY THIS SCRIPT

The crt.sh web UI is manual, unfiltered, uncacheable — raw cert records with no classification and no local storage.

  • Classifies: direct / wildcard / SAN leak
  • Deduplicates across cert records
  • Caches to SQLite — repeat queries instant
  • Outputs structured JSON for piping
  • Zero pip deps — pure Python stdlib

> SAN LEAKS — WHAT ARE THEY

A Subject Alternative Name (SAN) is the list of hostnames a cert covers. When a CA bundles multiple domains into one cert, those other domains appear as SAN entries — leaking the server's other tenants.

Querying nmap.org reveals seclists.org, sectools.org, insecure.org — all hosted on the same infrastructure. One CT query, entire server cluster mapped.
